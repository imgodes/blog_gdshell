Existem diversas barreiras para começar no pentest de aplicações iOS, dentre elas podemos listar: 
- a linguagem de programação do iOS (Swift e Objective C) não é algo ensinado de forma tão difundida quanto Java do android por exemplo.
- é necessário ter um dispositivo físico (iPhone) em uma versão da qual o jailbreak seja possível (vamos falar sobre isso mais tarde).
- é altamente recomendado ter um MacOS, pois o uso de grande parte das ferramentas e interações com o iPhone serão mais fáceis por meio dele.
- não recomendo utilizar ambientes virtualizados, **eu** não tive boa experiência com eles.
- Se você conseguiu um iPhone antigo, certifique-se de não instalar updates de SO. Assim como vimos nas proteções contra Jailbreak em [iOS Basics](/docs/category/ios-basics-1), qualquer atualização provavelmente vai invalidar todo seu Jailbreak.

Então confie em mim, se você não possui os requisitos acima, não vale a pena comprar nenhum curso. Na minha experiência, mesmo utilizando os dispositivos certos e seguindo a risca os manuais e documentações, já foi difícil, imagine sem eles.
